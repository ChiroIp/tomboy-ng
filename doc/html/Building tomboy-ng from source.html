Building tomboy-ng from source

You can build tomboy-ng on Windows (tested on 10), Mac (tested on Sierra and El Capitan) and Linux (tested on Ubuntu Mate 16.04, Ubuntu GNOME 18.04, Lubuntu 18.04, and Manjaro 17.1.0). There are binary install kits for Linux, Windows and Mac. If you want to build your own (or, please, help with development) that too is easy. Requires Free Pascal, Lazarus and the KControls component to build. I have been working with Lazarus 1.8, just released late 2017. If you run tomboy-ng within the debugger, remember that there are several handled exceptions, the debugger insists on telling you about them so make sure you press "continue". Outside the debugger, you will not be aware of this happening. You will need:
  • An install of Lazarus and Free Pascal. See here for a guide: http://wiki.freepascal.org/Installing_Lazarus
  • The Bitbucket version of KControls installed to your Lazarus: https://bitbucket.org/tkweb/kcontrols/downloads/
Preparing to build git clone https://github.com/tomboy-notes/tomboy-ng.git cd tomboy-ng/tomboy-ng Next, depending on your operating system, you'll build in one of the following ways: Building x86_64 Linux lazbuild $BUILDOPTS --cpu="x86_64" --build-mode=Release --os="linux" Tomboy_NG.lpi Building i386 Linux lazbuild $BUILDOPTS --cpu="i386" --build-mode=ReleaseLin32 --os="linux" Tomboy_NG.lpi Building x86_64 Windows lazbuild $BUILDOPTS --cpu="x86_64" --build-mode=ReleaseWin64 --os="win64" Tomboy_NG.lpi Building i386 Windows lazbuild $BUILDOPTS --cpu="i386" --build-mode=ReleaseWin32 --os="win32" Tomboy_NG.lpi
